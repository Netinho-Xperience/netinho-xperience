Plan de Desarrollo MVP NX®

Versión: 1.0

Uso: Dirección de Proyecto, Desarrollo y Lanzamiento Controlado

Este documento define el alcance mínimo viable de la plataforma Netinho Xperience para lanzar una primera versión funcional, controlada y jurídicamente defendible.

1. Objetivo del MVP

Construir una primera versión de la plataforma NX que permita al usuario:

  • seleccionar un tipo de activo;
  • conocer claramente el alcance del servicio;
  • realizar el pago;
  • cargar evidencias y documentos;
  • recibir un informe digital;
  • validar el código único del informe.

2. Alcance del MVP

Carga de evidencias Sí

Historial completo de usuario No en MVP

3. Flujo MVP

Landing → Selección de activo → Modal informativo → Pago → Expediente → Carga de evidencias → Procesamiento → Informe → Validación pública

Vigencia del informe 15 días calendario desde la emisión.

6. Criterios de éxito del MVP

  • El usuario entiende qué compra antes de pagar.
  • El usuario conoce los plazos antes de pagar.
  • El usuario puede completar el proceso desde celular.
  • El usuario recibe un código único.
  • El usuario puede validar el informe.
  • El informe contiene descargos claros.
  • El proceso queda trazado.

7. Lo que NO se desarrollará en el MVP

  • Aplicación móvil nativa.
  • Marketplace de especialistas.
  • Automatización completa avanzada.
  • Historial extendido del cliente.
  • Programa social visible.
  • Dashboard corporativo avanzado.
  • Integraciones complejas con múltiples entidades.

8. Principio de lanzamiento

El MVP debe ser simple, funcional, mobile-first, legalmente prudente y suficiente para validar si el usuario peruano está dispuesto a pagar por una herramienta de apoyo a la toma de decisiones patrimoniales.

9. Próxima fase después del MVP

  • Mejorar automatización.
  • Agregar historial de usuario.
  • Agregar certificados de eliminación de datos.
  • Integrar nuevas fuentes de información.
  • Crear biblioteca educativa NX.
  • Escalar el modelo hacia productos premium.
