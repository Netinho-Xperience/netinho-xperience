Mapa de Estados y Flujos NX®

Versión: 1.0

Uso: Desarrollo, operación, soporte, pruebas y trazabilidad

Este documento define los estados oficiales del expediente NX, los flujos principales, las excepciones operativas y las respuestas del sistema ante escenarios normales, incompletos, fallidos o contingentes.

1. Principio operativo

Todo expediente NX debe tener un estado claro, trazable y consultable. Ninguna evaluación debe quedar sin estado definido.

Regla PMO: cada cambio de estado debe generar un registro en logs_expediente.

2. Estados oficiales del expediente

Estado Descripción Responsable del cambio
pago_pendiente El usuario inició el flujo, pero el pago aún no fue validado. Sistema / Niubiz
activo El pago fue aprobado y el expediente fue creado. Sistema
carga_pendiente El expediente está habilitado, pero aún faltan evidencias o documentos. Usuario / Sistema
carga_parcial El usuario cargó algunos elementos, pero no alcanzó el mínimo requerido. Sistema
carga_completa El expediente alcanzó el mínimo operativo para análisis. Sistema
en_analisis Las evidencias están siendo procesadas. Sistema / IA
observado El expediente requiere corrección, recarga o información adicional. Sistema
informe_generando El informe está siendo estructurado y convertido a PDF. Sistema
informe_emitido El informe fue generado y está disponible. Sistema
vencido_carga El usuario no completó la carga dentro del plazo habilitado. Sistema
vencido_informe El informe superó su vigencia de 15 días calendario. Sistema
rechazado El expediente fue rechazado por causa técnica, integridad, riesgo o uso indebido. Sistema / Administración
anulado El expediente o informe fue anulado por error, reclamo, duplicidad o causa justificada. Administración
reembolso_en_revision Existe una solicitud de reembolso pendiente de evaluación. Soporte
reembolsado El pago fue revertido conforme a la política aplicable. Soporte / Pasarela

4. Flujo de pago aprobado

Paso Evento Estado resultante
1 Usuario acepta condiciones y continúa al pago. pago_pendiente
2 Niubiz confirma pago aprobado. activo
3 Sistema crea código único NX. activo
4 Sistema habilita 48 horas de carga. carga_pendiente
5 Usuario ve pantalla de bienvenida. carga_pendiente

5. Flujo de pago rechazado

Evento Respuesta del sistema
Pago rechazado por pasarela. Mostrar mensaje claro y permitir nuevo intento.
No existe confirmación de pago. No crear expediente activo.
Usuario consulta soporte. Solicitar código de operación si lo tuviera.

6. Flujo de pago aprobado pero expediente no creado

Este es un escenario crítico y debe estar cubierto por soporte y política de reembolso.
Acción Respuesta
Pago aprobado sin expediente. Registrar incidente técnico.
Usuario contacta soporte. Validar operación en pasarela.
Pago confirmado. Crear expediente manual o automático tardío.
No se puede habilitar servicio. Evaluar reembolso conforme a política.

7. Flujo de carga incompleta

Escenario Respuesta
Usuario carga solo parte de los documentos. Estado: carga_parcial.
Faltan elementos mínimos. Mostrar checklist y porcentaje de avance.
Usuario decide continuar con lo disponible. Permitir solo si alcanza mínimo operativo.
No alcanza mínimo operativo. No iniciar análisis. Solicitar información adicional.

8. Flujo de evidencia insuficiente

Escenario Respuesta
Foto borrosa. Marcar como insuficiente y pedir nueva foto.
PDF ilegible. Marcar como observado.
Documento incompleto. Registrar información faltante.
Archivo no permitido. Rechazar carga e indicar formato aceptado.

9. Flujo de vencimiento de carga

Condición Respuesta
Transcurren 48 horas sin carga completa. Estado: vencido_carga.
Usuario cargó parcialmente. El expediente queda cerrado o sujeto a revisión según reglas vigentes.
Usuario solicita reactivación. Evaluar nueva activación o nuevo pago según política.
Usuario reclama reembolso. Aplicar Política de Reembolsos.

10. Flujo de análisis

Paso Descripción Estado
1 Expediente alcanza mínimo operativo. carga_completa
2 Sistema inicia procesamiento. en_analisis
3 Gemini organiza hallazgos. en_analisis
4 Sistema calcula indicadores. en_analisis
5 Sistema estructura informe. informe_generando
6 PDF se emite. informe_emitido

11. Flujo de falla de IA

Escenario Respuesta
Gemini no responde. Reintentar procesamiento.
Gemini devuelve análisis insuficiente. Registrar limitación y usar nivel de incertidumbre.
Archivo no procesable. Marcar evidencia como observada.
Falla persistente. Informar demora, registrar incidencia y activar contingencia.

12. Flujo de informe emitido

Evento Resultado
PDF generado. Estado: informe_emitido.
Código único creado. Validación pública habilitada.
Fecha de vencimiento asignada. 15 días calendario.
Usuario descarga informe. Registrar log informe_descargado.

13. Flujo de vencimiento del informe

Condición Respuesta
Transcurren 15 días desde emisión. Estado: vencido_informe.
Usuario valida código vencido. Mostrar informe vencido.
Usuario requiere actualización. Crear nuevo expediente o nueva evaluación.

14. Flujo de validación pública

Entrada Respuesta
Código válido y vigente. Mostrar estado: informe auténtico y vigente.
Código válido pero vencido. Mostrar estado: informe auténtico, pero vencido.
Código inexistente. Mostrar estado: código no encontrado.
Informe anulado. Mostrar estado: informe anulado.

15. Flujo de reclamo

Evento Respuesta
Usuario registra reclamo. Generar código único de reclamo.
Reclamo asociado a expediente. Registrar vínculo en logs.
Reclamo por resultado no esperado. Responder con alcance del servicio, descargos y evidencia del proceso.
Reclamo por falla técnica. Evaluar logs, pago, estado y posible reembolso.

16. Flujo de reembolso

Escenario Respuesta
Pago validado pero acceso no habilitado. Puede proceder reembolso.
Usuario tuvo acceso y dejó vencer plazo. No procede reembolso ordinario.
Informe emitido. No procede reembolso ordinario.
Falla técnica comprobable. Evaluar reembolso total o parcial.

18. Reglas de comunicación al usuario

Situación Mensaje recomendado
Pago aprobado Gracias por confiar en NX. Tu expediente ha sido creado.
Carga pendiente Aún faltan elementos para completar tu expediente.
Evidencia insuficiente La evidencia cargada no permite un análisis suficiente. Sube una nueva versión siguiendo la guía.
Expediente vencido El plazo de carga ha vencido. Puedes iniciar una nueva evaluación si deseas continuar.
Informe emitido Tu Informe de Decisión Patrimonial NX® está listo.
Informe vencido El informe fue emitido correctamente, pero su vigencia ha expirado.

19. Regla final de control

Ningún expediente debe avanzar a informe_emitido sin código único, fecha de emisión, fecha de vencimiento, indicadores, descargos y registro de log.
