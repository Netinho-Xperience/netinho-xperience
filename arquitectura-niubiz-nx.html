Arquitectura Niubiz NX®

Versión: 1.0

Uso: Pagos, activación de expediente, trazabilidad y experiencia de usuario

Este documento define la arquitectura funcional y operativa para integrar Niubiz como pasarela de pago dentro de la plataforma Netinho Xperience.

1. Objetivo

Permitir que el usuario pague el servicio NX de forma segura, clara y trazable, activando automáticamente un expediente digital con plazo definido para cargar información.

2. Precio inicial del servicio

S/ 49.00 por Informe de Decisión Patrimonial NX®.

El precio podrá actualizarse según evolución del servicio, costos operativos, promociones o cambios del modelo comercial.

3. Principio UX antes del pago

Antes de redirigir al usuario a Niubiz, la plataforma deberá mostrar un modal de confirmación con:

  • nombre del servicio;
  • precio final;
  • plazo para cargar información;
  • plazo estimado de emisión;
  • vigencia del informe;
  • alcance del servicio;
  • limitaciones principales;
  • enlaces a términos, privacidad, descargos y reembolsos.
Regla UX: el usuario debe entender qué está comprando antes de pagar.

4. Plazos operativos asociados al pago

Elemento Plazo
Acceso para cargar información 48 horas desde validación del pago.
Emisión del informe Hasta 48 horas desde recepción completa de evidencias.
Vigencia del informe 15 días calendario desde emisión.
Conservación de evidencias 90 días.
Conservación del informe 24 meses.

5. Flujo funcional de pago

Fase Descripción
1. Usuario inicia evaluación Selecciona vehículo o inmueble.
2. Modal previo al pago Se muestra precio, alcance, plazos y descargos principales.
3. Aceptación Usuario acepta condiciones mínimas y continúa.
4. Redirección a Niubiz Usuario realiza el pago en entorno seguro.
5. Validación de pago Niubiz retorna resultado de operación.
6. Creación de expediente Si el pago es aprobado, NX crea código único.
7. Activación de acceso Se habilita carga por 48 horas.
8. Notificación Se muestra mensaje de bienvenida y datos del expediente.

6. Estados del pago

Estado Significado Acción NX
pendiente Pago iniciado, aún sin confirmación. No crear expediente activo.
aprobado Pago validado correctamente. Crear expediente y habilitar carga.
rechazado Pago no autorizado. Informar al usuario y permitir nuevo intento.
error Falla técnica o respuesta incompleta. Registrar log y mostrar canal de soporte.
reembolsado Pago revertido según política. Actualizar expediente y bloquear acceso si corresponde.

7. Datos mínimos del pago

Campo Uso
id_pago Identificador interno.
id_expediente Relación con expediente NX.
monto Monto pagado.
moneda PEN.
estado_pago Pendiente, aprobado, rechazado, error o reembolsado.
codigo_operacion Código de operación de pasarela.
fecha_pago Fecha y hora del pago.
pasarela Niubiz.

8. Creación de expediente posterior al pago

Cuando el pago sea aprobado, la plataforma deberá generar un código único.

Formato sugerido:
NX-AAAAMMDD-HHMMSS-XXXX

Ejemplo:

NX-20260623-224522-1937

Este código deberá utilizarse para soporte, seguimiento, informe, validación pública y reclamos.

9. Mensaje de bienvenida post-pago

Gracias por confiar en Netinho Xperience.

Tu expediente ha sido creado correctamente.

Tendrás 48 horas para cargar la información solicitada. Te guiaremos paso a paso para que completes tu expediente con claridad y seguridad.

10. Temporizador visible

Después del pago, el usuario deberá ver un contador:

Acceso habilitado:
47h 59m restantes

El contador reduce ansiedad, mejora la finalización del proceso y evita reclamos por vencimiento.

11. Manejo de errores

Escenario Respuesta recomendada
Pago rechazado Informar con claridad y permitir nuevo intento.
Pago aprobado pero expediente no creado Registrar log, mostrar soporte y aplicar política de revisión.
Usuario cierra la pantalla Enviar correo con enlace para continuar, si el pago fue aprobado.
Pasarela no responde Estado pendiente, log y reintento de validación.

12. Reglas de reembolso vinculadas a pago

El sistema deberá respetar la Política de Reembolsos NX.

  • Si el pago fue validado y nunca se habilitó el acceso, podrá proceder reembolso.
  • Si el usuario tuvo acceso y dejó vencer el plazo, no procede reembolso ordinario.
  • Si el informe ya fue emitido, no procede reembolso ordinario.
  • Si el resultado no coincide con expectativas del usuario, no procede reembolso.

13. Seguridad

  • No exponer claves privadas de Niubiz en frontend.
  • Validar pagos desde servidor o función segura.
  • Registrar logs de cada evento de pago.
  • Evitar crear expedientes activos sin confirmación real de pago.
  • Proteger contra doble activación por pago repetido.
  • Guardar código de operación para trazabilidad.

14. Logs obligatorios

  • pago_iniciado;
  • pago_aprobado;
  • pago_rechazado;
  • pago_error;
  • expediente_creado;
  • acceso_habilitado;
  • reembolso_solicitado;
  • reembolso_aprobado;
  • reembolso_rechazado.

15. Texto recomendado antes del botón de pago

Al continuar, aceptas que el pago habilitará un expediente digital NX por 48 horas para cargar información. El informe será emitido hasta 48 horas después de recibir las evidencias completas y tendrá una vigencia de 15 días calendario.

16. Enlaces obligatorios en pantalla de pago

  • Términos y Condiciones
  • Política de Privacidad
  • Descargos Legales
  • Política de Reembolsos
  • Centro de Ayuda
  • Libro de Reclamaciones

17. Recomendación MVP

En la primera versión, el objetivo no debe ser construir una pasarela perfecta y compleja. El objetivo es asegurar un flujo claro, trazable y defendible: usuario informado, pago validado, expediente creado, temporizador visible, carga habilitada e informe emitido.
