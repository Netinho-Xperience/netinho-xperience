Diccionario de Datos NX®

Versión: 1.0

Uso: Desarrollo, Supabase, formularios, validaciones, informes, pagos y trazabilidad

Este documento define los campos principales utilizados por Netinho Xperience para usuarios, expedientes, pagos, evidencias, indicadores, informes, validación pública y logs.

1. Principios del Diccionario de Datos

  • Usar nombres claros y consistentes.
  • Evitar duplicidad innecesaria.
  • Definir obligatoriedad de cada campo.
  • Definir formato, longitud y validaciones.
  • Reducir datos personales al mínimo necesario.
  • Permitir trazabilidad completa del expediente.
  • Facilitar automatización futura con Supabase, Gemini y Niubiz.

2. Convenciones generales

Convención Descripción
uuid Identificador único interno generado por base de datos.
text Texto libre controlado por validaciones de frontend y backend.
timestamptz Fecha y hora con zona horaria.
numeric Valor numérico, usado especialmente para montos.
integer Número entero, usado para indicadores de 0 a 100.
boolean Valor verdadero/falso.

3. Tabla: profiles

Campo Tipo Obligatorio Descripción Validación
id uuid Sí ID del usuario asociado a Supabase Auth. Autogenerado.
nombres text Sí Nombres del solicitante. 2 a 80 caracteres.
apellidos text Sí Apellidos del solicitante. 2 a 100 caracteres.
tipo_documento text Sí DNI, CE, Pasaporte u otro. Lista cerrada.
numero_documento text Sí Número de documento del solicitante. DNI: 8 dígitos. CE/Pasaporte: 6 a 12 caracteres.
correo text Sí Correo de contacto del usuario. Formato email válido.
created_at timestamptz Sí Fecha de creación del perfil. Autogenerado.

4. Tabla: expedientes

Campo Tipo Obligatorio Descripción Validación
id uuid Sí Identificador interno del expediente. Autogenerado.
codigo text Sí Código único NX del expediente. Formato: NX-AAAAMMDD-HHMMSS-XXXX.
user_id uuid Sí Usuario propietario del expediente. Debe existir en auth.users.
tipo_activo text Sí Vehículo o inmueble. Valores permitidos: vehiculo, inmueble.
estado text Sí Estado operativo del expediente. Lista cerrada de estados NX.
fecha_pago timestamptz No Fecha de pago aprobado. Se completa al validar pago.
fecha_activacion timestamptz No Fecha de activación del expediente. Debe generarse después del pago aprobado.
fecha_vencimiento_carga timestamptz No Fecha límite para cargar evidencias. 48 horas desde activación.
fecha_emision timestamptz No Fecha de emisión del informe. Se completa al emitir PDF.
fecha_vencimiento_informe timestamptz No Fecha de vencimiento del informe. 15 días calendario desde emisión.

5. Estados permitidos del expediente

Estado Significado
pago_pendiente El pago fue iniciado, pero aún no fue aprobado.
activo El pago fue aprobado y el expediente fue habilitado.
carga_pendiente Faltan documentos o evidencias mínimas.
carga_completa El expediente alcanzó el mínimo requerido.
en_analisis El expediente está siendo procesado.
informe_emitido El informe fue generado.
vencido El plazo de carga o vigencia del informe expiró.
rechazado El expediente fue rechazado por riesgo, integridad o imposibilidad técnica.

6. Tabla: pagos

Campo Tipo Obligatorio Descripción Validación
id uuid Sí Identificador interno del pago. Autogenerado.
expediente_id uuid Sí Expediente asociado al pago. Debe existir.
monto numeric Sí Monto pagado. Inicialmente 49.00.
moneda text Sí Moneda del pago. PEN.
pasarela text Sí Pasarela utilizada. Niubiz u otra autorizada.
estado_pago text Sí Estado del pago. pendiente, aprobado, rechazado, error, reembolsado.
codigo_operacion text No Código de operación de la pasarela. Debe guardarse si existe.
fecha_pago timestamptz No Fecha y hora del pago aprobado. Se completa al aprobar pago.

7. Tabla: evidencias

Campo Tipo Obligatorio Descripción Validación
id uuid Sí Identificador interno de la evidencia. Autogenerado.
expediente_id uuid Sí Expediente asociado. Debe existir.
tipo_evidencia text Sí Ejemplo: fachada, tablero, odómetro, copia literal, boleta informativa. Lista según tipo de activo.
tipo_archivo text Sí Foto, PDF, video u otro. Valores controlados.
nombre_archivo text Sí Nombre original del archivo. Registrar para trazabilidad.
storage_path text Sí Ruta en Supabase Storage. Bucket privado.
estado_revision text Sí Estado de revisión de la evidencia. recibido, procesado, rechazado, insuficiente.
observacion text No Comentario sobre calidad o problema detectado. Máximo recomendado: 500 caracteres.

8. Tabla: indicadores

Campo Tipo Obligatorio Descripción Validación
riesgo_nx integer Sí Índice de Riesgo NX®. 0 a 100.
incertidumbre_nx integer Sí Índice de Incertidumbre NX®. 0 a 100.
preparacion_nx integer Sí Índice de Preparación para Decidir NX®. 0 a 100.
calidad_expediente integer Sí Calidad del Expediente NX®. 0 a 100.
confianza_informe integer Sí Nivel de Confianza del Informe NX®. 0 a 100.

9. Tabla: informes

Campo Tipo Obligatorio Descripción Validación
id uuid Sí Identificador interno del informe. Autogenerado.
expediente_id uuid Sí Expediente asociado. Debe existir.
codigo_validacion text Sí Código único de validación pública. Único.
version text Sí Versión del informe. Ejemplo: 1.0.
url_pdf text Sí Ruta segura del PDF. Bucket privado con URL firmada.
estado text Sí Estado del informe. vigente, vencido, anulado.
fecha_emision timestamptz Sí Fecha de emisión. Autogenerada al emitir.
fecha_vencimiento timestamptz Sí Fecha de vencimiento. 15 días calendario.

10. Tabla: logs_expediente

Campo Tipo Obligatorio Descripción Validación
id uuid Sí Identificador del log. Autogenerado.
expediente_id uuid Sí Expediente asociado. Debe existir.
evento text Sí Acción registrada. Lista controlada de eventos.
detalle text No Detalle del evento. Máximo recomendado: 1000 caracteres.
created_at timestamptz Sí Fecha y hora del evento. Autogenerado.

11. Eventos mínimos de log

  • pago_iniciado
  • pago_aprobado
  • pago_rechazado
  • expediente_creado
  • acceso_habilitado
  • archivo_cargado
  • archivo_rechazado
  • carga_completa
  • analisis_iniciado
  • informe_emitido
  • informe_descargado
  • informe_vencido
  • reclamo_registrado

12. Reglas generales de validación

Elemento Regla
DNI 8 dígitos.
Correo Formato email válido.
Indicadores Solo enteros de 0 a 100.
Código NX Debe ser único.
Estado Debe pertenecer a lista cerrada.
Archivo Debe cumplir tipo, peso y formato permitido.

13. Datos no recomendados para mostrar públicamente

  • Número completo de DNI.
  • Documentos cargados.
  • Fotos originales.
  • Dirección exacta del inmueble.
  • Datos completos del solicitante.
  • Información sensible del expediente.

14. Datos permitidos en validación pública

  • Código del informe.
  • Estado: vigente, vencido o no encontrado.
  • Fecha de emisión.
  • Fecha de vencimiento.
  • Tipo de activo.
  • Nombre parcialmente anonimizado.
  • Versión del informe.

15. Conservación de datos

Tipo de dato Plazo
Evidencias y documentos 90 días.
Informes 24 meses.
Códigos de validación 24 meses.
Logs 24 meses.
