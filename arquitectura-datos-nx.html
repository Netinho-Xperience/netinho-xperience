Arquitectura de Datos NX® (ADB-NX)

Versión: 1.0

Uso: Desarrollo, Base de Datos y Operación

Este documento define las entidades de información que soportan la Plataforma de Inteligencia Patrimonial Netinho Xperience.

Principios

  • Minimización de datos.
  • Trazabilidad.
  • Seguridad.
  • Escalabilidad.
  • Integridad.
  • Eliminación programada.

Tabla: Usuarios

Campo Descripción
id_usuario Identificador interno.
nombres Nombres completos.
apellidos Apellidos completos.
tipo_documento DNI, CE, Pasaporte u otro.
numero_documento Número de documento.
correo Correo electrónico.
fecha_registro Fecha de creación.

Tabla: Expedientes

Campo Descripción
id_expediente Código único NX.
id_usuario Relación con usuario.
tipo_activo Vehículo o Inmueble.
estado Abierto, en análisis, emitido, vencido.
fecha_pago Fecha de pago.
fecha_activacion Inicio del acceso.
fecha_vencimiento_carga Fin de las 48 horas.
fecha_emision Fecha del informe.
fecha_vencimiento_informe 15 días.

Tabla: Pagos

Campo Descripción
id_pago Identificador.
id_expediente Relación con expediente.
monto S/49.00 u otro.
estado_pago Pendiente, validado, rechazado.
fecha_pago Timestamp.
pasarela Niubiz.

Tabla: Evidencias

Campo Descripción
id_evidencia Identificador.
id_expediente Relación.
tipo_archivo Foto, PDF, video.
nombre_archivo Nombre original.
fecha_carga Timestamp.
estado_revision Recibido, procesado, rechazado.

Tabla: Indicadores NX®

Campo Descripción
riesgo_nx 0-100.
incertidumbre_nx 0-100.
preparacion_nx 0-100.
calidad_expediente 0-100.
confianza_informe 0-100.

Tabla: Informes

Campo Descripción
id_informe Identificador.
codigo_validacion Código único.
url_pdf Ubicación segura.
fecha_emision Timestamp.
vigente Sí o No.
version 1.0, 2.0, etc.

Tabla: Logs

Campo Descripción
id_log Identificador.
id_expediente Relación.
fecha Timestamp.
evento Acción realizada.
detalle Información adicional.
